# Build with us: Support and Knowledge Base for Beginners

Welcome! This repository holds everything needed to plan, prepare for, and run **Build with us: Support and Knowledge Base for Beginners** — a hands-on lab where organizations build a working AI agent grounded in their own documents, then use AI-assisted coding to build a small app around it.

If you're new here, start with [What This Program Is](#what-this-program-is), then use the [Repository Map](#repository-map) to find the exact document you need.

---

## What This Program Is

**Build with us: Support and Knowledge Base for Beginners** is a **2-day hands-on lab** (9:00 AM – 3:00 PM each day) for developers, technical business users, citizen builders, IT admins, and company teams — many of whom are **not** senior developers.

- **Day 1** — Build and evaluate a **Technical Support / Knowledge Base agent** in Azure AI Foundry, grounded in company-provided documents.
- **Day 2** — Use **GitHub Copilot and VS Code** to build or modify a small app that uses the Day 1 agent — learning to build *with AI assistance*.

### Scope for This Round

**In scope:** Technical Support agents, Knowledge Base agents, troubleshooting agents, internal knowledge discovery, support-documentation search, agents grounded in company documents, a simple app that uses the agent, and AI-assisted coding with GitHub Copilot.

**Out of scope:** General AI strategy sessions, broad custom enterprise AI, complex production deployment architecture, large-scale app modernization, advanced agent orchestration, and full production security/compliance review.

## Program Flow

The program runs from registration through a two-day build event:

1. **Registration** — Organizations submit a contact, use case, desired outcome, and attendee list; approved orgs join the Build with us Teams community.
2. **Kickoff (Week 1)** — Combined with the first readiness meeting; homework and pre-skilling are assigned.
3. **Weekly Readiness Meetings (Weeks 1–4)** — Clear business and technical blockers as a collaborative community.
4. **Final Readiness Review (1 week out)** — Go / No-Go check on business, technical, and participation requirements.
5. **The Event** — Day 1 builds the agent; Day 2 builds the app.

See the full step-by-step [High Level Timeline](01-Program-Overview/High%20Level%20Timeline.md).

## Readiness at a Glance

An organization is **ready** for the event when all three tracks are complete:

- **Business readiness** — Problem, desired outcome, success criteria, example questions, and knowledge-base documents are defined and available.
- **Technical readiness** — Azure subscription, Azure AI Foundry access, VS Code, Python, GitHub account, and GitHub Copilot Pro (or higher) are all verified and working.
- **Pre-skilling** — The Microsoft AI Co-Innovation Lab Getting Started curriculum is complete.

Organizations with unresolved critical blockers may be deferred to a future program so event time stays focused on building — not environment setup.
