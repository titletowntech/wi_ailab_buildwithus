Build with us: Support & Knowledge Base for Beginners

Welcome to Build with us: Support and Knowledge Base for Beginners! Before the event, each organization completes this homework so you arrive ready to build — not set up. Your progress will be reviewed together during the weekly readiness meetings, where we help each other clear any blockers.

How to Use This Package

  1. Work through the three parts below: Business Readiness, Technical Readiness, and Pre-Skilling.
  2. Fill in the two worksheets (links below).
  3. Bring your status and any blockers to each weekly readiness meeting.
  4. Post questions and share discoveries in the Build with us Teams community.

Part 1: Business Readiness

Define a Technical Support or Knowledge Base use case using the Business Readiness Worksheet.

Complete the following:

  • ☐ One-paragraph use case statement (the problem you want to solve)
  • ☐ Desired business outcome
  • ☐ Success criteria
  • ☐ Who will use the solution, and roughly how many users
  • ☐ Sample user questions (that should work and should not work)
  • ☐ Sample documents and data for your knowledge base
  • ☐ Rank your event goals (1–5) so we know what success looks like for you

Part 2: Technical Readiness

Get your environment ready using the Technical Readiness Worksheet. Step-by-step help for each item is on the pre-skilling site.

Complete the following:

  • ☐ Azure subscription access — guide
  • ☐ Azure AI Foundry access — guide
  • ☐ VS Code installed — guide
  • ☐ Python installed — guide
  • ☐ GitHub account ready — guide
  • ☐ GitHub Copilot Pro or higher subscription — guide

If any item needs IT or admin approval at your company, start that process early and let us know at the next readiness meeting so we can plan around the timing.

Part 3: Pre-Skilling

Complete the learning curriculum in the Pre-Skilling Package before the event. It covers how our lab works, the key AI terms and concepts, and a live agent example.

  • ☐ Pre-skilling complete (see the Pre-Skilling Package)

What to Bring to Each Weekly Readiness Meeting

  • Which items are done and confirmed working
  • Which items you could not complete, and what happened when you tried
  • Anything waiting on IT or admin approval
  • Any questions about which account or plan to use

Definition of Ready

You're ready for the event when:

  • Your business use case, outcome, success criteria, questions, and documents are complete
  • Your Azure, Azure AI Foundry, VS Code, Python, GitHub, and Copilot setup is verified
  • The Getting Started pre-skilling is complete
  • No critical blockers remain
