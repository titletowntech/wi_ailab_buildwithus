Build with us: Support & Knowledge Base for Beginners

A high-level, chronological timeline of every step in the program, from initial registration through the end of the two-day event.

Phase 1: Registration (Pre-Program)

  1. Organizations register — Submit primary point of contact, use case description, desired business outcome, and expected attendee list.
  2. Applications reviewed — Organizations are approved and accepted into the program.
  3. Onboarding begins — Approved organizations receive onboarding communications.
  4. Teams community access — Participants are added to the Build with us Teams community (central hub for announcements, readiness support, questions, file sharing, and peer collaboration).

Phase 2: Kickoff Meeting (Week 1)

The Kickoff Meeting is combined with the first weekly readiness meeting.

  1. Program overview — Establish expectations, explain how the program works, and clarify pre-event requirements.
  2. Introductions — Participants meet each other and the collaboration model is introduced.
  3. Agenda and milestones — Review event agenda, milestones, and participation requirements.
  4. Homework assigned — Business and technical readiness artifacts are assigned.
  5. Pre-skilling assigned — Participants directed to the Microsoft AI Co-Innovation Lab Getting Started curriculum.

Phase 3: Homework & Pre-Skilling (Weeks 1–4)

Business Readiness Homework

  1. Write a one-paragraph use case statement.
  2. Define the desired business outcome.
  3. Establish success criteria.
  4. Identify sample user questions.
  5. Gather sample documents and data.

Technical Readiness Homework

  1. Verify Azure subscription access.
  2. Verify Azure AI Foundry access.
  3. Install VS Code.
  4. Install Python.
  5. Confirm GitHub account readiness.
  6. Activate GitHub Copilot Pro or higher.

Pre-Skilling

  1. Complete the Microsoft AI Co-Innovation Lab Getting Started curriculum.

Phase 4: Weekly Readiness Meetings (Weeks 1–4)

Objective: Eliminate business and technical barriers before event week.

Each weekly meeting follows a standard agenda:

  1. Pre-skilling review — Check curriculum progress.
  2. Technical readiness review — Resolve environment and setup blockers.
  3. Business readiness review — Refine use case, outcomes, and documents.
  4. Collective learning — Share discoveries, lessons learned, emerging use cases, and open questions across the community.

Phase 5: Final Readiness Review — Go / No-Go (One Week Before Event)

  1. Confirm pre-skilling complete.
  2. Confirm business readiness complete — Problem, outcome, success criteria, example questions, and documents ready.
  3. Confirm technical readiness complete — Azure, VS Code, Python, and GitHub Copilot validated.
  4. Confirm participation requirements — Attendance at kickoff and readiness meetings, and confirmed event attendees.
  5. Identify unresolved blockers — Organizations with critical unresolved blockers may be deferred to a future program.

Phase 6: The Event — Day 1 (9:00 AM – 3:00 PM)

Focus: Build and evaluate a Knowledge Base / Technical Support Agent using Azure AI Foundry.

  1. Verify Azure & Foundry access — Confirm subscription and Azure AI Foundry project readiness.
  2. Prepare knowledge base documents — Load company-provided manuals, troubleshooting guides, SOPs, and articles.
  3. Build the agent — Create a knowledge-based agent grounded in the provided documents.
  4. Test example questions — Validate questions that should and should not work.
  5. Evaluate the agent — Assess accuracy, grounding, and troubleshooting quality against success criteria.

Phase 7: The Event — Day 2 (9:00 AM – 3:00 PM)

Focus: Build software with AI assistance using GitHub Copilot and VS Code.

  1. Set up development environment — Confirm VS Code, Python, and GitHub Copilot are working.
  2. Connect the agent — Integrate the Day 1 agent into a simple application experience.
  3. Build with GitHub Copilot — Use AI-assisted coding to build or modify an application that uses the agent.
  4. Test the application — Validate the app works with the agent.
  5. Wrap-up — Review outcomes and discuss plans for future deployment.
