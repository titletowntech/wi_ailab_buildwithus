Build with us: Support and Knowledge Base for Beginners — Organizer Preparation Checklist

Build with us: Support & Knowledge Base for Beginners

An organizer's checklist of everything I need to do to prepare for and run the event — from setting up registration through final Go/No-Go and event-day logistics. Items already completed are checked.

1. Program Setup & Materials

  • ☑ Define program scope (Technical Support & Knowledge Base agents only) — README.md
  • ☑ Define the overall program flow (registration → kickoff → weekly meetings → Go/No-Go → event) — High Level Timeline.md
  • ☐ Plan the event timeline and all key dates (work backward from the event date) — Event Planning Timeline Worksheet.md
  • ☑ Create the Business Readiness Worksheet template — Business Readiness Worksheet.md
  • ☑ Create the Technical Readiness Worksheet template — Technical Readiness Worksheet.md
  • ☑ Finalize the 2-day event agenda (Day 1: build & evaluate agent; Day 2: build app with Copilot) — Day 1 Agenda.md, Day 2 Agenda.md
  • ☑ Prepare the homework package to hand out at kickoff — Homework Package.md
  • ☑ Prepare the pre-skilling package to hand out at kickoff — Pre-Skilling Package.md
  • ☑ Create the Day 1 facilitator guide from the Day 1 Agenda.md — Day 1 Facilitator Guide.md
  • ☑ Prepare the Day 1 guided-agent and good-vs-bad-data sample files — Sample Files - Good, Sample Files - Bad
  • ☑ Write the Day 1 guided-agent prompt and test questions (should-work / should-not-work) — Day 1 Guided Agent Kit.md
  • ☑ Prepare the Day 2 template application — using the Wisconsin AI Co-Innovation Lab Prototype Template (titletowntech/wi_ailab_prototype_template)
  • ☑ Create the Day 2 facilitator guide from the Day 2 Agenda.md — Day 2 Facilitator Guide.md
  • ☑ Write the Day 2 guided-build script, connection recipe, and Copilot prompts — Day 2 Guided Agent Kit.md
  • ☑ Prepare the Day 2 Copilot 101 handout (sample Markdown and prompt examples) — Copilot 101 - Sample Markdown and Prompts.md

2. Pre-Skilling Site

  • ☑ Publish the Getting Started curriculum site
  • ☑ Publish the Technical Readiness Overview page
  • ☑ Publish the Azure Subscription Readiness page
  • ☑ Publish the Azure AI Foundry Readiness page
  • ☑ Publish the VS Code Readiness page
  • ☑ Publish the Python Readiness page
  • ☑ Publish the GitHub Copilot Readiness page
  • ☐ Review all pages for plain-language, non-expert tone

3. Registration

  • ☐ Set up the registration process / form (contact, use case, outcome, attendee list)
  • ☐ Define acceptance criteria for organizations
  • ☐ Review and approve applicants into the program
  • ☐ Send acceptance / onboarding communications
  • ☐ Set up the Build with us Teams community — Teams Community Setup Guide.md
  • ☐ Add approved organizations to the Teams community
  • ☐ Configure Teams channels (announcements, readiness support, Q&A, file sharing, blocker tracking) — Teams Community Setup Guide.md

4. Kickoff Meeting (Week 1)

  • ☐ Schedule the kickoff / first readiness meeting
  • ☐ Prepare the program overview deck
  • ☐ Prepare agenda, milestones, and participation requirements
  • ☐ Prepare the community collaboration model explanation
  • ☐ Distribute the homework package
  • ☐ Distribute the pre-skilling package
  • ☐ Facilitate participant introductions

5. Weekly Readiness Meetings (Weeks 1–4)

  • ☐ Schedule the 4 weekly readiness meetings
  • ☑ Prepare a standard agenda (pre-skilling, technical, business, collective learning) — Weekly Readiness Meeting Agenda.md
  • ☐ Track each organization's pre-skilling progress
  • ☐ Track each organization's technical readiness status
  • ☐ Track each organization's business readiness status
  • ☐ Maintain a shared blocker log with issue, owner, and status
  • ☐ Facilitate cross-team knowledge sharing
  • ☐ Assign highest-priority action items each week

6. Final Readiness Review — Go / No-Go (1 Week Before Event)

  • ☐ Schedule the final readiness review
  • ☐ Confirm each org's pre-skilling is complete
  • ☐ Confirm each org's business readiness is complete
  • ☐ Confirm each org's technical readiness is complete
  • ☐ Confirm participation requirements met (kickoff + readiness meetings)
  • ☐ Confirm final event attendee lists
  • ☐ Make Go / No-Go decision per organization
  • ☐ Communicate deferrals for orgs with unresolved critical blockers

7. Event Logistics

  • ☐ Confirm venue / virtual platform and room setup
  • ☐ Confirm event dates and daily schedule (9:00 AM – 3:00 PM, both days)
  • ☐ Confirm facilitators / coaches for hands-on support
  • ☐ Confirm catering / breaks (if in person)
  • ☐ Test lab environment end-to-end (Azure AI Foundry + template app)
  • ☐ Prepare printed or digital handouts for both days
  • ☐ Prepare backup plan for common blockers (Azure access, Copilot, Python)

8. Event Delivery

Day 1 — Build & Evaluate the Agent

  • ☐ Confirm participants can access Azure & Azure AI Foundry
  • ☐ Guide participants to load their knowledge base documents
  • ☐ Guide participants to build a knowledge/technical support agent
  • ☐ Guide participants to test example questions
  • ☐ Guide participants to evaluate the agent against success criteria

Day 2 — Build the App with GitHub Copilot

  • ☐ Confirm VS Code, Python, and Copilot are working for each participant
  • ☐ Guide participants to connect the agent to a simple app
  • ☐ Guide participants to build/modify the app with Copilot
  • ☐ Guide participants to test the app
  • ☐ Facilitate wrap-up and future deployment discussion
